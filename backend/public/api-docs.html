Overview

The Ultimate Buggy Lapping Tournament API provides comprehensive endpoints for managing tournament data, player statistics, zone information, and race results. All responses follow a Strapi-compatible JSON format for consistency.

Base URL: http://localhost:1337 (development)

Content-Type: application/json

Response Format: Strapi-compatible JSON with data wrapper

Authentication

Admin endpoints require authentication via an admin token passed in the Authorization header.

Header Format

Authorization: Bearer YOUR_ADMIN_TOKEN

⚠️ Security Note: Admin endpoints modify critical tournament data. Ensure your admin token is kept secure and never committed to version control.

Setting Admin Token

Set the ADMIN_TOKEN environment variable on your server:

export ADMIN_TOKEN="your-secure-token-here"

Tournaments

GET /api/tournaments

Retrieve the current tournament information including title, dates, prizes, rules, and registration details.

Response Example

{
  "data": {
    "id": 1,
    "attributes": {
      "title": "Ultimate Buggy Lapping Championship",
      "description": "8-week championship series",
      "start_date": "2025-01-15T00:00:00.000Z",
      "end_date": "2025-03-12T00:00:00.000Z",
      "is_active": true,
      "total_weeks": 8,
      "prize_info": "Prize pool details...",
      "rules": "Tournament rules and regulations...",
      "registration_info": "How to register...",
      "stream_url": "https://...",
      "discord_url": "https://...",
      "points_config": { ... }
    }
  }
}
PUT /api/tournaments/:id 🔒 Admin

Update tournament information. Requires admin authentication.

Path Parameters

Parameter Type Description
id integer Tournament ID

Request Body

{
  "title": "Updated Tournament Title",
  "description": "New description",
  "start_date": "2025-01-20T00:00:00.000Z",
  "end_date": "2025-03-20T00:00:00.000Z",
  "prize_info": "Updated prize information",
  "rules": "Updated rules"
}

Players

Retrieve all players with their statistics and standings.

    {

      "attributes": {
        "name": "Player Name",
        "zone": "Zone 1",
        "status": "active",

        "races_participated": 5,

        "best_finish_time_ms": 154567,
        "position": 1

    }

POST /api/players 🔒 Admin

Create a new player.

Request Body

Field Type Required Description
name string Required Player name
zone string Optional Home zone
status string Optional Player status (default: "active")
PUT /api/players/:id 🔒 Admin

Update an existing player's information.

DELETE /api/players/:id 🔒 Admin

Delete a player. Also removes all associated race results.

Race Results

GET /api/race-results

Retrieve all race results with player and zone relationships.

Response Example

{
  "data": [
    {
      "id": 1,
      "attributes": {
        "position": 1,
        "points_earned": 10,
        "lap_time": "2:34.567",
        "finish_time": "2:34.567",
        "finish_time_ms": 154567,
        "race_date": "2025-01-15T00:00:00.000Z",
        "player": {
          "data": {
            "id": 1,
            "attributes": { "name": "Player Name" }
          }
        },
        "zone": {
          "data": {
            "id": 1,
            "attributes": { "name": "Zone 1" }
          }
        }
      }
    }
  ]
}
POST /api/race-results 🔒 Admin

Create a new race result. Automatically updates player statistics.

Request Body

Field Type Required Description
player_id integer Required Player ID
zone_id integer Required Zone ID
finish_time string Required Time in MM:SS.mmm format
position integer Optional Finishing position (auto-calculated if not provided)
race_date string Optional ISO 8601 date string
POST /api/race-results/bulk 🔒 Admin

Create multiple race results at once with automatic position calculation based on finish times.

Request Body

Field Type Required Description
zone_id integer Required Zone ID for the race
week_number integer Optional Week number (default: 1)
race_date string Optional ISO 8601 date string
results array Required Array of result objects
results[].player_id integer Required Player ID
results[].finish_time string Required Time in MM:SS.mmm format
results[].notes string Optional Notes for this result

Example Request

{
  "zone_id": 1,
  "week_number": 3,
  "race_date": "2025-01-20T00:00:00.000Z",
  "results": [
    {
      "player_id": 1,
      "finish_time": "01:23.456"
    },
    {
      "player_id": 2,
      "finish_time": "01:25.789"
    },
    {
      "player_id": 3,
      "finish_time": "01:22.123"
    }
  ]
}

Note: Positions are automatically calculated based on finish times. The fastest time gets position 1, second fastest gets position 2, etc.

PUT /api/race-results/:id 🔒 Admin

Update a race result. Recalculates affected player statistics.

DELETE /api/race-results/:id 🔒 Admin

Delete a race result. Recalculates affected player statistics.
